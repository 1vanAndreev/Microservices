# Microservices

Учебный проект по построению **микросервисной архитектуры** на локальной машине с использованием Docker Compose, RabbitMQ, Prometheus, Grafana и GitHub Actions.

## О проекте

Проект представляет собой прототип платформы **BlogFlow** — системы для управления пользователями, публикацией постов и доставкой контента в микросервисной архитектуре.

Цель проекта — получить практический опыт в:
- разбиении системы на отдельные сервисы,
- контейнеризации приложений,
- локальном развертывании инфраструктуры,
- настройке взаимодействия сервисов,
- базовом мониторинге и CI/CD.

## Архитектура

Система состоит из нескольких компонентов:

- **ApiGateway** — единая точка входа в систему
- **UsersService** — управление пользователями
- **PostsService** — работа с постами
- **DeliveryService** — доставка / обработка контента
- **SQL Server** — основное хранилище данных
- **Prometheus** — сбор метрик
- **Grafana** — визуализация метрик

## Что реализовано

В проекте реализованы:
- разбиение приложения на несколько независимых сервисов;
- контейнеризация сервисов через Dockerfile;
- локальное развертывание всей системы через Docker Compose;
- API Gateway для маршрутизации запросов;
- базовый мониторинг сервисов через Prometheus и Grafana;
- CI/CD pipeline в GitHub Actions для сборки, тестирования и публикации Docker-образов.
